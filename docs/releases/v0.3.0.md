# mcpkit v0.3.0

Released 2026-06-29.

Minor release. The version moves to 0.3.0 because the SEP-2907 error-code
renumber is wire-breaking for clients that switch on the numeric JSON-RPC
error code (see Breaking). The bulk of the work hardens the SEP-2575
stateless wire to dispatch parity with the legacy path and rounds out the
OAuth step-up / lazy-scope flow.

Forward-looking: 0.3.x is the SEP-2577 deprecation window. Roots, sampling,
and logging surfaces carry `// Deprecated:` blocks with no behavior change;
removal is targeted for v0.4.

## Breaking

- Error codes renumbered for SEP-2907: `HeaderMismatch` -32001 → **-32020**,
  `MissingRequiredClientCapability` -32003 → **-32021**. Clients that switch
  on the numeric code must update; clients matching on the error message or
  `data.reason` are unaffected. Both wires and the stateless dispatcher track
  the new values. (PR 813)

## Stateless wire (SEP-2575)

- Middleware that short-circuits with `*core.AuthError` now surfaces as
  HTTP 403 + `WWW-Authenticate` on the stateless wire, instead of being
  folded into `-32603` / HTTP 200. As a parity consequence, a non-`AuthError`
  middleware error now maps to HTTP 401 (matching the legacy wire) rather
  than `-32603`. (PR 816, issue 815)
- SEP-2356 file-input validation now runs on the stateless `tools/call` path
  before the handler, matching the legacy dispatcher. (PR 834)
- Client emits the SEP-2243 `Mcp-Name` routing header for task operations
  (`tasks/get` / `tasks/update` / `tasks/cancel`) on the stateless wire. (PR 832)

## Auth (ext/auth)

- `OAuthTokenSource` defers scope acquisition until a server challenge selects
  the scope. Standalone `Token()` returns `core.ErrNoTokenYet` until the source
  is armed by a challenge or the caller pins explicit `Scopes`. Fixes the eager
  PRM bug where a per-method-gated server saw the client request the broad PRM
  scope first. (PR 820, issue 818)
- Client honors the `WWW-Authenticate scope=` value on a 401 retry per
  RFC 6750 §3.1; every scope-aware 401 routes through `TokenForScopes`, so a
  scope-less 401 still arms the source via the PRM `scopes_supported` fallback
  instead of looping. (PR 819)
- Scope-challenge 403s advertise the `resource_metadata` link so clients can
  discover the PRM document from the challenge. (PR 819)
- `AcceptedScopes` OR-hierarchy + `includeGrantedScopes` opt-in exercised in a
  new `step-up-keycloak` SUT with `tests/keycloak` integration. (PR 819, PR 822)

## Examples

- `examples/common` gains a `--wire` flag for SEP-2575 wire selection, adopted
  across the non-UI examples so each walkthrough can run on either wire. (PR 826, PR 828)
- Dual-mode audit + `make verify-dual` harness drives example walkthroughs over
  both the legacy and stateless wires to catch dispatch-parity regressions.
  See `examples/DUAL_MODE_AUDIT.md`. (issue 478, PR 836)
- `step-up-keycloak` no longer forces stateless mode by default. (PR 821)

## Conformance

- New external stateless-draft checker report: the mcpkit client graded by an
  independent third-party gauntlet on the integrated stateless draft wire
  (`2026-07-28`). Run via `make testconf-external-checker`; output in
  `conformance/EXTERNAL_CHECKER.md`. (PR 830)
- Tasks v2 (SEP-2663) and MRTR (SEP-2322) suites retargeted at
  `modelcontextprotocol/conformance` `main` directly — both merged upstream,
  so the fork is no longer needed for them.
- Upstream audit grades the SEP-2663 tasks scenarios against the
  `examples/tasks-v2` fixture.

## Events (experimental)

- `eventId` is now globally unique (random) rather than a per-source counter.
- `whole-enchilada/events` demo cleanup: cross-replica delivery surfacing
  (`X-Replica` per delivery), streamer reconnect on replica drop, and a Step 4
  curl that sends the full stateless `_meta` envelope. (PR 831)

## Docs / infra

- `CAPABILITIES.md` protocol-negotiation line corrected — `2026-07-28` is
  preferred and `2025-03-26` is also negotiated; the stateless-wire version
  list is advertised separately.
- `scripts/verify-submodule-deps.sh` discovers sub-modules dynamically,
  fixing the stale `ext/protogen` path (moved to `experimental/ext/protogen`)
  that broke `bump-root`, `audit`, and `ci`.
- Version → 0.3.0; `bump-root` points every sub-module's root require at v0.3.0.
