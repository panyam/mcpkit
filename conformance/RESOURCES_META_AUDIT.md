# apps/compat fixture parity audit

Per-fixture wire-surface parity between the mcpkit-Go drop-in and upstream's TypeScript reference server. Generated by `scripts/apps_meta_audit.py`; runs the same `apps_parity_diff.py` that gates DOCKER-mode CI in `apps-playwright-docker-inner.sh`.

Surfaces compared per fixture:

1. `initialize` — `serverInfo` + fixture-relevant `capabilities`
2. `tools/list` — every tool's name, title, description, schemas, tool `_meta`
3. `resources/list` — registered resource URIs + def `_meta`
4. `resources/templates/list` — URI templates
5. `resources/read` — per-content `_meta` (where `csp` + `permissions` live)

Iframe HTML body (`text`/`blob`) is intentionally NOT compared — different bundlers produce different bytes for the same logical App.

**Summary:** 0 pass · 2 actionable drift · 19 systematic-only drift · 0 error · 21 total.

**Actionable drift** = the audit found a fixture-specific divergence (missing `_meta`, wrong URI, etc.) that's worth fixing. **Systematic-only drift** = mcpkit-framework-wide convention differences that fire on every fixture and aren't fixture bugs — listed in the *Known systematic differences* section below.

## Known systematic differences

These appear on every (or nearly every) fixture and are mcpkit-framework conventions vs upstream's per-fixture conventions, not fixture bugs:

- **`capabilities.prompts.listChanged`** — mcpkit's server framework advertises the prompts capability on every server. Upstream advertises it only when the fixture registers a prompt. None of the compat fixtures register prompts, so all 21 show this in the initialize diff. Server-framework follow-up.
- **`resources/list` — `name`** — mcpkit auto-derives `"<tool> UI"` from the AppToolConfig's `cfg.Name`. Upstream uses the URI as the name. Different conventions; neither is wrong; both are spec-compliant.
- **`resources/list` — `description`** — upstream's fixtures pass a description through `registerAppResource`; mcpkit's `AppToolConfig` doesn't surface a `Description` field for the resource def, so mcpkit omits it. Add the field to make per-fixture descriptions reachable.
